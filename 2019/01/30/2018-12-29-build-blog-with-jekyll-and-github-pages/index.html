基于Jekyll和GitHub Pages搭建个人博客

发表于 2019-01-30 | 分类于 文章教程

本文介绍如何利用Jekyll创建一个静态站点，并托管到GitHub Pages上。

JeKyll

JeKyll是一个基于Ruby的博客类静态网站生成器。

Jekyll是一个简单，可扩展的静态站点生成器。您可以使用自己喜欢的标记语言编写文本，然后通过布局来创建静态网站。在整个过程中，您可以调整网站URL的显示方式，布局中显示的数据等

基于JeKyll创建静态网站

  1. 安装Ruby开发环境 (以macOS 为例) macOS Mojave 10.14 可省略以下安装步骤

    1.1 安装Homebrew和Ruby

    1.2 导出Ruby环境变量

    1.3 检查Ruby是否安装成功

  2. 安装打包器和jekyll

  3. 创建一个新的Jekyll网站

    若提示 command not found: jekyll ，需要把gem路径配置到PATH里面

  4. 编译网站并启动本地服务

    访问http://localhost:4000/查看效果

GitHub Pages

GitHub Pages是直接从GitHub仓库托管的个人网站和项目网站。

基于GitHub Pages托管网站

  1. 创建一个仓库
    前往GitHUb创建一个新的仓库，仓库名称为username.github.io，其中username是你的GitHub用户名或者组织名称。

  2. 克隆仓库
    把步骤一创建的仓库克隆到本地。

  3. 创建第一个页面
    进入项目目录，新建一个index.html的文件。

  4. 推送到远程仓库
    增加、提交和推送你的更改

  5. 你已经成功完成
    访问https://username.github.io查看效果

整合部署

将Jekyll生成的静态网站复制到username.github.io仓库并提交推送到GitHub

后面直接修改仓库文件，用Jekyll编译提交即可。
